# 这个神器让 NotebookLM 效率翻倍，研究党必备！

如果你是 NotebookLM 的重度用户，一定遇到过这样的痛点：看到好文章想收藏，看到 YouTube 视频想整理，读到 arXiv 论文想做笔记——但每次都要手动复制粘贴，切换窗口，重复操作。

今天要分享的这个开源插件，彻底解决了这个问题。

## 一键发送，告别重复劳动

**Send to NotebookLM** 是一个浏览器扩展，核心功能简单粗暴：在任何网页上点一下，内容直接进入你的 NotebookLM。

支持的内容类型包括：

- 普通网页文章

- arXiv 学术论文

- YouTube 视频

不需要复制粘贴，不需要手动导入，一个按钮搞定。这对于需要大量收集资料的研究者、学生、内容创作者来说，节省的不只是时间，更是那种被琐碎操作打断思路的烦躁感。

## 真正的杀手功能：完整的工作空间

如果只是一键发送，这个插件也就是个效率工具。但当你打开它的管理界面，会发现这完全是一个**付费级别的专业工具**——而且还是免费开源的。

![笔记本管理界面](https://cdn.gooo.ai/user-files/7feec78653e5575cc517ed584830dc04177c9d4bd01054c2049cb5b4c0d58f71@chat)

### 统一的笔记本管理中心

插件提供了一个完整的「NotebookLM 工作空间」，你可以在这里：

- **批量导入**：一次性导入多个 URL，不用一个个手动添加

- **笔记本管理**：查看所有笔记本，搜索、筛选、快速定位

- **一键生成多种内容**：每个笔记本都可以直接生成幻灯片、信息图、报告、视频概览

看到那些按钮了吗？「幻灯片」「信息图」「报告」「视频概览」——这些都是 NotebookLM 原生需要手动操作的功能，现在在插件里点一下就能触发。

### Enhancer：自动化你的工作流

更厉害的是 **Enhancer** 功能。它可以在 NotebookLM 生成内容时，自动注入你预设的提示词。

这意味着什么？假设你经常需要：

- 提炼论文的核心观点和创新点

- 总结视频的时间轴和关键信息

- 生成符合特定格式的读书笔记

传统做法是每次手动输入这些要求。而有了 Enhancer，你可以把这些常用指令保存为预设，之后 NotebookLM 生成内容时会自动按照你的偏好处理。

这不是简单的自动化，而是让工具真正「记住」你的工作方式。

## 为什么值得一试

这个插件的设计哲学很清晰：**减少摩擦，增强控制**。

它不试图替代 NotebookLM，而是让你更高效地使用 NotebookLM。插件本身提供了一个选项页面，可以管理所有收集的内容，触发各种生成任务，配置 Enhancer 设置。

技术栈也很现代：基于 WXT、Svelte 和 Tailwind CSS 构建，代码开源在 GitHub 上。对于开发者来说，这意味着可以根据自己的需求进行定制；对于普通用户来说，开源意味着透明和安全。

## 如何安装

目前插件还在开发阶段，安装方式有两种：

**方式一：直接安装**

1. 下载项目的 `.zip` 压缩包并解压

2. 打开 Chrome 浏览器，访问 `chrome://extensions/`

3. 开启右上角的“开发者模式”

4. 点击“加载已解压的扩展程序”

5. 选择解压后的文件夹（确保里面有 `manifest.json` 文件）

**方式二：从源码构建**

1. 克隆或下载 GitHub 仓库

2. 运行构建命令

3. 在 Chrome 扩展页面加载 `.output/chrome-mv3` 目录

安装后，打开 NotebookLM 登录，然后在任何支持的页面上点击插件按钮即可使用。

## 适合谁用

这个工具特别适合：

**学术研究者**：需要大量阅读论文、整理文献的人，可以快速将 arXiv 论文导入 NotebookLM，用 AI 辅助理解和总结。

**内容创作者**：需要从各种来源收集素材的人，可以把网页、视频统一管理，用 NotebookLM 生成创作灵感。

**终身学习者**：习惯用 NotebookLM 做知识管理的人，这个插件让收集和整理的流程更加顺畅。

## 一点思考

这个插件的出现，反映了一个趋势：AI 工具的价值不只在于功能本身，更在于如何融入用户的实际工作流程。

NotebookLM 很强大，但它的输入方式仍然有摩擦。Send to NotebookLM 做的事情看似简单，实际上是在解决“最后一公里”的问题——让强大的工具真正好用起来。

而 Enhancer 功能更进一步，它让工具开始“记住”你的使用习惯，这种个性化的自动化才是 AI 时代效率工具应该有的样子。
